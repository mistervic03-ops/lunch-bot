# 밥라투스트라 브랜드 가이드

## 현재 이름

제품의 이름은 **밥라투스트라**다.

익숙한 철학적 인물의 이름을 점심과 결합해, 과장되게 진지한 철학자가 구성원 앞에 오늘의 점심 선택지를 선언한다는 캐릭터를 만든다. 특정 철학자의 실제 사상이나 문장을 설명하거나 직접 인용하는 제품은 아니다.

Python 패키지, systemd unit, 배포 경로와 브랜드 전용 환경변수에는 영문 식별자 `bapratustra`를 사용한다. 배포 전 하드 전환했으므로 옛 내부 식별자의 호환 alias나 환경변수 fallback은 제공하지 않는다.

Slack 앱 이름은 `밥라투스트라`로 표시한다. Manifest의 봇 사용자 `display_name`은 Slack이 소문자 영문, 숫자, 하이픈, 밑줄과 마침표만 허용하므로 `bapratustra`를 사용한다. 공개 메시지 본문에서는 한글 제품명을 유지한다.

## 핵심 캐릭터

- 밥라투스트라는 점심의 정답을 맞히는 예언자가 아니라 선택할 세 갈래 길을 제시하는 철학자다.
- 캐릭터성은 공개 추천 메시지의 등장과 선언에 집중한다.
- 식당, 메뉴, 가격, 링크와 반응 방법은 빠르게 이해할 수 있게 평범하고 정확하게 쓴다.
- 오류와 운영 알림에서는 역할극을 하지 않고 발생 시각, 단계, 결과와 오류 ID를 사실적으로 전달한다.
- 구성원의 선택이나 취향을 평가하거나 특정 식당을 반드시 골라야 한다고 말하지 않는다.

## 대표 문구와 메시지

대표 문구는 다음과 같다.

```text
밥라투스트라는 이렇게 말했다.
```

후보가 세 곳일 때의 고정 선언은 다음과 같다.

```text
점심은 스스로 정해지지 않는다. 선택되어야 한다.
오늘 그대들 앞에는 세 갈래의 길이 놓여 있다.
```

전체 메시지는 다음 구조를 사용한다.

```text
📜 밥라투스트라는 이렇게 말했다.

점심은 스스로 정해지지 않는다. 선택되어야 한다.
오늘 그대들 앞에는 세 갈래의 길이 놓여 있다.

1. 마루식당 — 제육볶음
2. 소바집 — 냉소바
3. 대성집 — 순댓국

먹고 싶은 후보 번호에 반응해 주세요.
```

초기에는 대표 선언을 무작위로 바꾸지 않는다. 반복감이 실제 문제로 확인되면 검수된 소수의 선언문을 추가할지 다시 결정한다.

채널 온보딩은 캐릭터 인사 한 줄 뒤에 사용법을 평범하게 설명한다.

```text
📜 밥라투스트라의 점심 채널에 오신 것을 환영합니다.

평일 오전 11시(KST)에 점심 후보 세 곳을 올립니다.
먹고 싶은 후보 번호의 1️⃣, 2️⃣, 3️⃣ 반응을 눌러 주세요. 여러 곳을 골라도 됩니다.
새 후보는 아래 버튼에서 등록할 수 있고, 누적 좋아요와 순위는 ‘밥라투스트라의 전당’에서 확인할 수 있습니다.
```

## 후보 부족 문구

후보가 두 곳이면 `오늘 보이는 점심의 길은 둘뿐이다.`, 한 곳이면 `오늘 확인된 점심의 운명은 하나뿐이다.`라고 선언한다. 목록 뒤에는 다음의 평범한 안내만 표시한다.

```text
새 후보는 아래 버튼에서 등록할 수 있습니다.
```

## 말투와 문장부호 원칙

- 등장과 선언은 장엄하고 진지하게 쓰되 두 문장 이내로 제한한다.
- 후보 목록과 사용 안내는 친절하고 간결한 존댓말을 사용한다.
- 운영 메시지는 캐릭터 말투 없이 기술적으로 정확하게 쓴다.
- em dash(`—`)는 일반 문장이나 선언에 가급적 사용하지 않는다.
- 후보 행에서 식당과 메뉴 정보를 구분하는 `식당 — 메뉴` 형식에는 em dash를 사용할 수 있다.
- 브랜드 이모지는 제목의 `📜` 하나만 사용한다.
- 후보 선택용 `1️⃣`, `2️⃣`, `3️⃣`은 장식이 아니라 기능적 입력이므로 제목의 이모지 제한과 별도로 취급한다.

## 시각 콘셉트

Slack 앱 아이콘은 니체를 연상시키는 큰 콧수염과 진지한 철학자의 인상을 가진 가상의 밥라투스트라 캐릭터를 사용한다. 실제 니체 초상화를 그대로 복제하거나 표준적인 외형이 없는 차라투스트라를 종교적 인물처럼 직접 묘사하지 않는다. 점심 또는 밥을 연상시키는 요소를 결합하고, 작은 Slack 아이콘에서도 얼굴과 콧수염이 먼저 읽히도록 단순하게 표현한다.

이 아이콘은 Slack 앱 설정에 적용되어 있다. 현재 원본 이미지 파일은 저장소에서 관리하지 않으므로 이후 아이콘을 교체하거나 재현해야 한다면 사용한 원본 자산을 먼저 확보하고 저장소 포함 여부를 별도로 결정한다. Manifest의 앱 배경색은 `#523078`을 유지한다.

웹 리더보드의 브라우저와 화면 제목은 `밥라투스트라의 전당`으로 한다. 화면은 반복해서 사용하는 사내 도구이므로 콘텐츠의 우선순위를 브랜드 장식보다 앞세운다. 흰색 바탕에 보라색 `#523078`을 소수의 강조 요소에만 사용한다. 상단에는 옅은 보라색 편집형 히어로를 두고 왼쪽에 `우리의 점심 기록`, 화면 제목과 `좋아요와 추천 기록을 모았습니다.`를 표시하고 오른쪽에 니체 초상을 둔다. 후보 등록과 Sheet 관리 행동은 히어로 밖의 요약 영역에 배치해 초상과 겹치지 않게 한다. 그라데이션, 그림자, 카드형 대시보드, 고전풍 세리프, 금색, 한자와 영문 장식 문구는 사용하지 않는다. 메뉴 순위는 전체 너비의 주 콘텐츠로 두고, 식당과 추천인 순위는 그 아래에서 같은 너비로 배치한다. 제품명은 화면 제목 한 번에만 노출하고 다른 영역에서 반복하지 않는다. 순위, 좋아요, 추천 횟수와 추천인 후보 수는 평범하고 정확하게 표시한다. 순위와 추천 빈도의 관계는 `좋아요 순위는 추천 순서에 영향을 주지 않습니다.`처럼 사용자가 바로 이해할 수 있게 설명하고 `공정 순환 추천` 같은 내부 용어를 화면에 노출하지 않는다.

후보 등록 웹은 브랜드 소개 화면이 아니라 반복해서 쓰는 사내 도구로 표현한다. 상단 워드마크 한 번 외에는 캐릭터 문구를 사용하지 않고 `식당·메뉴 등록`, `등록된 후보`, `Google Sheet 관리`처럼 행동과 목적을 바로 알 수 있는 이름을 사용한다. 흰색과 옅은 회색 바탕, 얇은 경계, 6px 안팎의 작은 모서리와 보라색 주 버튼 하나로 위계를 만든다. 데스크톱 등록 폼은 두 열, 후보 목록은 표에 가까운 행 구조를 사용하고 모바일에서는 한 열로 전환한다. 활성 후보의 상태를 반복 표시하지 않으며 비활성 후보만 `추천 제외`로 표시한다.

두 웹 화면은 같은 Pretendard, 보라색 `#523078`, 44px 주 버튼, 14~16px 본문과 데이터 크기를 공유한다. 후보 등록의 주요 행동은 보라색 채움 버튼으로 표시한다. 읽기가 주목적인 전당의 등록 행동은 옅은 보라색 버튼으로 강도를 낮추고 Sheet 관리는 테두리 없는 링크로 표시한다. 반복되는 데이터 링크는 기본 밑줄을 제거하고 hover와 focus에서만 밑줄을 표시한다. 후보 웹은 도구형, 전당은 편집형이라는 각자의 역할은 유지하되 서로 다른 제품처럼 보이는 별도 색상·글꼴·버튼 체계를 만들지 않는다.

두 웹 화면의 브라우저 탭에는 `#523078` 바탕과 흰색 콧수염을 단순화한 같은 SVG favicon을 사용한다.

웹 폰트는 저장소에 포함한 Pretendard Variable을 직접 제공한다. 클라이언트에 설치된 폰트나 외부 CDN에 의존하지 않으며, 폰트 파일과 함께 원본 라이선스를 보관한다.

웹 히어로 우측에는 1885년경 니체의 공용 도메인 초상을 낮은 투명도의 흑백 워터마크로 한 번만 사용한다. 타원형 투명 마스크로 사진의 네 가장자리를 모두 흐리게 해 직사각형 경계가 보이지 않게 한다. 사진은 제목보다 먼저 보이지 않아야 하며 별도 콘텐츠나 프로필 이미지처럼 표현하지 않는다. 좁은 화면에서는 숨긴다. 원본과 출처·라이선스 정보는 `bapratustra/static/images/`에서 함께 관리한다.

## 아직 결정하지 않은 사항

- 실제 운영 채널의 표시 이름을 언제 변경할지
- 사용 중 반복감이 생겼을 때 선언문을 추가할지

이 항목들은 구현 전에 선택지와 트레이드오프를 검토하고 확정한다.
