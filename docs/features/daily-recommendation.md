# 정기 점심 추천

## 목적

밥괘는 점심 전용 옵트인 Slack 채널에 정해진 시간마다 점심 후보를 게시하여, 구성원이 메뉴를 고르고 함께 식사할 사람을 찾는 대화를 시작하도록 돕는다.

## 게시 일정

- 월요일부터 금요일까지 오전 11시(KST)에 게시한다.
- 대한민국 공휴일을 별도로 판별하지 않는다. 공휴일이 평일이면 게시 대상이다.
- 자동 메시지에 `@channel` 또는 `@here` 멘션을 사용하지 않는다.
- 개인 명령으로 자신만의 추천을 받는 기능은 초기 범위에 포함하지 않는다.

## 정상 추천

- 한 번에 서로 다른 식당 세 곳을 추천한다.
- 한 식당에 메뉴 행이 여러 개 있으면 그중 한 메뉴만 표시한다.
- 추천 후보를 식당별 카드로 나누지 않고 하나의 간결한 목록으로 표시한다.
- 브랜드 이모지는 남발하지 않는다. 초기 기준으로 제목의 `🔮` 하나만 사용한다.
- 메시지 게시 후 후보 순서에 맞는 `1️⃣`, `2️⃣`, `3️⃣` 반응을 봇이 추가한다. 번호 반응은 장식이 아니라 후보별 좋아요 입력이다.
- 가격, 지도 링크, 추천인이 있을 때만 해당 정보를 표시한다.
- 선택값이 없을 때 `정보 없음` 같은 대체 문구나 빈 구분자를 표시하지 않는다.

## 공정 순환 추천

복잡한 점수나 가중치를 사용하지 않고 추천 이력의 마지막 추천 시각만 이용한다.

1. 유효한 행을 식당 이름으로 묶는다.
2. 각 식당이 마지막으로 추천된 시각을 `recommendation_log`에서 찾는다.
3. 한 번도 추천되지 않은 식당을 가장 오래된 것으로 취급한다.
4. 마지막 추천 시각이 오래된 식당부터 최대 세 곳을 선택한다.
5. 마지막 추천 시각이 같으면 해당 식당들 사이에서 무작위로 순서를 정한다.
6. 선택된 식당에 메뉴가 여러 개라면 가장 오래 추천되지 않은 메뉴를 선택하고, 동률이면 무작위로 정한다.

식당과 메뉴를 비교할 때 앞뒤 공백과 연속된 공백만 정리한다. 별도의 내부 ID, 가중치, 점수, 추천 제외 기간은 도입하지 않는다.

식당명이 수정되면 과거 로그와 다른 식당으로 인식되어 우선 추천될 수 있다. 이는 구조를 단순하게 유지하기 위해 허용하는 한계이며, 직원은 같은 식당의 여러 메뉴 행에 같은 식당명을 사용한다.

## 같은 날짜의 중복 게시 방지

- 게시 전에 추천 로그를 읽어 현재 KST 날짜와 대상 Slack 채널 ID가 같은 행이 있는지 확인한다.
- 한 행이라도 있으면 해당 날짜와 채널의 게시가 완료된 것으로 보고 실행을 중단한다.
- 채널이 다르면 같은 날짜라도 별개의 실행으로 취급한다.
- 중복 방지는 Slack 게시 흐름을 구현할 때 실제 실행 경로에 연결한다.

## 게시와 로그 기록 순서

1. 후보와 기존 추천 로그를 읽고 검증한다.
2. 같은 KST 날짜와 채널의 기존 실행이 없는지 확인한다.
3. 추천을 계산하고 Slack 메시지를 게시한다.
4. 게시 성공 시 Slack 메시지 ID와 후보들을 추천 로그에 한 번에 추가한다.
5. 로그 추가가 실패하면 자동으로 Slack 게시를 재시도하지 않고 운영 오류로 알린다.

로그 기록에 실패한 뒤 수동 재실행하면 기존 로그만으로 중복을 판별할 수 없으므로, 운영자가 Slack 채널의 실제 게시 여부를 먼저 확인해야 한다. 별도의 예약 상태나 실행 테이블은 초기 복잡성을 줄이기 위해 두지 않는다.

메시지의 개념 예시는 다음과 같다. 최종 문구와 Slack 서식은 아직 확정하지 않았다.

```text
🔮 오늘의 밥괘가 나왔습니다.

1. 마루식당 — 제육볶음
   약 10,000원 · 추천: 철수 · 지도 보기

2. 소바집 — 냉소바
   지도 보기

3. 대성집 — 순댓국
   약 10,000원 · 추천: 영희

마음에 드는 후보 번호에 반응해주세요.

점괘를 믿을지는 여러분의 몫입니다.
```

## 후보가 부족할 때

### 유효한 식당이 한두 곳인 경우

- 존재하는 후보만 게시한다.
- 후보가 세 곳보다 적다는 사실을 간결하게 알린다.
- 구성원이 Google Sheet에 새로운 후보를 추가하도록 안내한다.

개념 예시는 다음과 같다.

```text
🔮 오늘의 밥괘는 아직 흐릿합니다.

1. 마루식당 — 제육볶음
2. 소바집 — 냉소바

새로운 점심 운명을 시트에 보태주세요.
```

### 유효한 식당이 없는 경우

- 사용자용 점심 채널에는 메시지를 게시하지 않는다.
- 비공개 `#밥괘-운영` 채널에 운영 오류를 알린다.

## 운영 오류 알림

- 비공개 `#밥괘-운영` 채널을 사용한다.
- 봇과 실제 운영 담당자만 채널에 참여한다.
- Slack의 Channel Manager 역할을 API로 자동 탐색하여 수신자를 정하지 않는다.
- 사용자용 점심 채널에는 기술 오류를 노출하지 않는다.
- 같은 오류에 대해 운영 담당자 각각에게 중복 DM을 보내지 않는다.

다음 상황은 운영 채널 알림 대상이다.

- Google Sheet를 읽지 못해 추천을 만들 수 없음
- 유효한 추천 후보가 없음
- Slack 추천 메시지 전송이 최종적으로 실패함
- 추천 이력 기능을 도입한 이후, 메시지는 게시됐지만 이력 기록에 실패함

운영 메시지에는 인증 정보, 토큰 또는 불필요한 내부 데이터를 포함하지 않는다. 실패 시각, 실패 단계, 최종 결과와 추적 가능한 오류 식별자만 간결하게 표시한다.

## 아직 결정하지 않은 사항

- 작업 실패 시 재시도 횟수와 간격
- 회사 자체 휴무일을 처리할 필요가 있는지
