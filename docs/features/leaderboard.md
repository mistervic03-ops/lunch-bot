# 밥라투스트라의 전당

## 목적과 범위

`밥라투스트라의 전당`은 Google Sheet에 쌓인 추천과 좋아요를 사내 구성원이 읽기 편한 형태로 보여주는 읽기 전용 웹페이지다. 기존 `인기 메뉴` Sheet 탭을 없애거나 일일 공정 순환 추천에 영향을 주지 않는다.

첫 버전은 다음만 제공한다.

- 전체 기간 기준 상위 메뉴 10개
- 전체 기간 기준 상위 식당 10개
- 현재 활성 후보를 가장 많이 보탠 추천인 10명
- 좋아요 마지막 집계 시각
- 공동 편집 Google Sheet로 이동하는 링크
- 애플리케이션 상태 확인을 위한 `/healthz`

기간 필터, 로그인, 관리자 기능, 데이터 수정, 별도 DB와 외부 공개는 포함하지 않는다.

## 지표와 정렬

### 사랑받은 메뉴

`recommendation_log`를 `식당 + 메뉴`로 묶어 좋아요 합계와 추천 등장 횟수를 계산한다. 좋아요 내림차순, 식당명과 메뉴명 오름차순으로 정렬한다. 현재 후보에서 비활성화되거나 삭제된 과거 메뉴도 기록에 남는다.

### 식당의 길

`recommendation_log`를 식당으로 묶어 모든 메뉴의 좋아요와 등장 횟수를 합산한다. 좋아요 내림차순, 식당명 오름차순으로 정렬한다.

### 길을 보탠 사람

현재 유효한 `lunch_options`의 `추천인`별 행 수를 센다. 추천인이 비어 있는 후보는 제외한다. 자유 입력값이므로 이 수치는 정확한 인사 평가나 보상 기준이 아니며, 화면에서도 후보를 보탠 수로만 표현한다.

모든 순위는 동점이어도 화면 행 순서대로 연속된 번호를 표시한다.

## 데이터와 캐시

- 웹 프로세스는 읽기 전용 Google 인증으로 `lunch_options`와 `recommendation_log`를 읽는다.
- 첫 요청 때 데이터를 읽고 같은 프로세스에서 5분간 캐시한다.
- 캐시 만료 후 첫 요청 한 건만 새 데이터를 읽으며 나머지 요청은 같은 결과를 사용한다.
- 한 번 정상 데이터를 읽은 뒤 Google Sheet 갱신이 일시 실패하면 직전 스냅샷을 5분 더 제공한다.
- 최초 읽기부터 실패하면 기술 정보를 노출하지 않는 브랜드 오류 화면과 HTTP `503`을 반환한다.
- `/healthz`는 Google Sheet를 호출하지 않는 프로세스 liveness 검사이며 `{"status":"ok"}`를 반환한다.

좋아요는 기존 평일 일일 작업에서 동기화되므로 웹페이지의 5분 캐시가 Slack 반응을 실시간 집계한다는 의미는 아니다.

## 접근과 운영

- 사내 네트워크 또는 사내 VPN에서만 접근 가능한 서버 포트로 제공한다.
- 애플리케이션 로그인과 외부 공개 URL을 만들지 않는다.
- CORS API나 브라우저 JavaScript 데이터 호출 없이 서버가 완성된 HTML을 반환한다.
- FastAPI, Jinja 템플릿과 정적 CSS만 사용하며 프론트엔드 빌드 도구는 두지 않는다.
- systemd `bapratustra-leaderboard.service`가 Uvicorn 단일 worker를 상시 실행한다.
- 공용 환경 파일을 사용하되 systemd가 Slack 토큰과 채널 설정을 웹 프로세스에 전달하지 않으며, 웹 프로세스의 `.env` 재로딩도 비활성화한다.
- 기본 포트는 `8030`이며 배포 전에 다른 내부 서비스와 충돌하지 않는지 확인한다.

운영 채널과 Sheet에서 리더보드로 연결하는 방식은 실제 운영 채널과 내부 접속 주소를 확정한 뒤 적용한다.

## 말투와 화면

- 제목은 `밥라투스트라의 전당`을 사용한다.
- 캐릭터 문구는 첫 화면 소개에만 쓰고 지표명과 숫자는 정확하게 표현한다.
- `최고`, `가장 좋은 식당`처럼 데이터가 보장하지 않는 평가 표현을 사용하지 않는다.
- 보라색, 종이색과 금색을 기본 색으로 사용하되 Slack 메시지의 이모지 규칙을 웹 장식에 기계적으로 적용하지 않는다.
- 작은 화면에서도 가로 스크롤 없이 순위와 핵심 수치를 읽을 수 있어야 한다.
