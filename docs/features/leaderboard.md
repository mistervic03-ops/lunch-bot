# 밥라투스트라의 전당

## 목적과 범위

`밥라투스트라의 전당`은 Google Sheet에 쌓인 추천과 좋아요를 사내 구성원이 읽기 편한 형태로 보여주는 읽기 전용 웹페이지다. Sheet의 별도 리더보드 탭을 두지 않으며 일일 공정 순환 추천에 영향을 주지 않는다.

첫 버전은 다음만 제공한다.

- 전체 기간 기준 상위 메뉴 10개
- 전체 기간 기준 상위 식당 10개
- 현재 활성 후보를 가장 많이 보탠 추천인 10명
- 좋아요 마지막 집계 시각
- 공동 편집 Google Sheet 관리 링크
- 후보 간편 등록 웹으로 이동하는 주 버튼
- 현재 활성 후보에 지도 링크가 있는 메뉴와 식당의 지도 링크
- 애플리케이션 상태 확인을 위한 `/healthz`

기간 필터, 로그인, 관리자 기능, 데이터 수정, 별도 DB와 외부 공개는 포함하지 않는다.

## 지표와 정렬

### 인기 메뉴

`recommendation_log`를 `식당 + 메뉴`로 묶어 좋아요 합계와 추천 횟수를 계산한다. 좋아요 내림차순, 식당명과 메뉴명 오름차순으로 정렬한다. 현재 후보에서 비활성화되거나 삭제된 과거 메뉴도 기록에 남는다.

현재 활성 `lunch_options`에 동일한 `식당 + 메뉴`와 지도 링크가 있으면 메뉴명을 해당 링크로 연결한다. 과거 로그에만 남은 메뉴이거나 링크가 비어 있으면 일반 텍스트로 표시한다.

### 인기 식당

`recommendation_log`를 식당으로 묶어 모든 메뉴의 좋아요와 추천 횟수를 합산한다. 좋아요 내림차순, 식당명 오름차순으로 정렬한다.

현재 활성 `lunch_options`에서 동일한 식당의 지도 링크를 찾으면 식당명을 연결한다. 같은 식당에 서로 다른 링크가 여러 개면 Sheet에서 먼저 읽은 링크를 사용한다. 링크가 없으면 일반 텍스트로 표시한다.

### 추천인

현재 유효한 `lunch_options`의 `추천인`별 행 수를 센다. 추천인이 비어 있는 후보는 제외한다. 자유 입력값이므로 이 수치는 정확한 인사 평가나 보상 기준이 아니며, 화면에서도 후보를 보탠 수로만 표현한다.

모든 순위는 동점이어도 화면 행 순서대로 연속된 번호를 표시한다.

## 데이터와 캐시

- 웹 프로세스는 읽기 전용 Google 인증으로 `lunch_options`와 `recommendation_log`를 읽는다.
- 첫 요청 때 데이터를 읽고 같은 프로세스에서 5분간 캐시한다.
- 캐시 만료 후 첫 요청 한 건만 새 데이터를 읽으며 나머지 요청은 같은 결과를 사용한다.
- 한 번 정상 데이터를 읽은 뒤 Google Sheet 갱신이 일시 실패하면 직전 스냅샷을 5분 더 제공한다.
- 최초 읽기부터 실패하면 기술 정보를 노출하지 않는 브랜드 오류 화면과 HTTP `503`을 반환한다.
- `/healthz`는 Google Sheet를 호출하지 않는 프로세스 liveness 검사이며 `{"status":"ok"}`를 반환한다.

좋아요는 기존 평일 일일 작업에서 동기화되므로 웹페이지의 5분 캐시가 Slack 반응을 실시간 집계한다는 의미는 아니다.

## 접근과 운영

- 사내 네트워크 또는 사내 VPN에서만 접근 가능한 서버 포트로 제공한다.
- 애플리케이션 로그인과 외부 공개 URL을 만들지 않는다.
- CORS API나 브라우저 JavaScript 데이터 호출 없이 서버가 완성된 HTML을 반환한다.
- FastAPI, Jinja 템플릿과 정적 CSS만 사용하며 프론트엔드 빌드 도구는 두지 않는다.
- systemd `bapratustra-leaderboard.service`가 Uvicorn 단일 worker를 상시 실행한다.
- 공용 환경 파일을 사용하되 systemd가 Slack 토큰과 채널 설정을 웹 프로세스에 전달하지 않으며, 웹 프로세스의 `.env` 재로딩도 비활성화한다.
- 기본 포트는 `8030`이며 배포 전에 다른 내부 서비스와 충돌하지 않는지 확인한다.

운영 채널과 Sheet에서 리더보드로 연결하는 방식은 실제 운영 채널과 내부 접속 주소를 확정한 뒤 적용한다.

## 말투와 화면

- 브라우저와 화면 제목은 `밥라투스트라의 전당`을 사용하고 제품명을 다른 영역에서 반복하지 않는다.
- 캐릭터 문구는 첫 화면 소개에만 쓰고 지표명과 숫자는 정확하게 표현한다.
- 상단 소개는 `우리의 점심 기록`과 `좋아요와 추천 기록을 모았습니다.`를 사용한다.
- `식당·메뉴 등록`은 보라색 주 버튼으로 후보 웹에 연결하고 `Google Sheet 관리`는 같은 높이의 외곽선형 보조 버튼으로 표시한다.
- `최고`, `가장 좋은 식당`처럼 데이터가 보장하지 않는 평가 표현을 사용하지 않는다.
- 흰색과 옅은 회색을 기본으로 사용하고 보라색은 순위와 버튼 등 소수의 강조 요소에만 사용한다.
- 옅은 보라색 단색 배경과 얇은 상하 구분선으로 편집형 히어로를 구성한다. 제목은 왼쪽, 니체 초상은 오른쪽에 두고 행동 버튼은 히어로 밖의 요약 영역에 배치한다. 메뉴 순위는 전체 너비의 주 콘텐츠로, 식당과 추천인 순위는 아래의 동일 너비 보조 콘텐츠로 표현한다.
- 히어로 우측에는 공용 도메인 니체 초상을 낮은 투명도의 흑백 워터마크로 배치한다. 타원형 투명 마스크로 사진의 네 가장자리를 흐리게 하고 760px 이하에서는 숨긴다.
- 1위의 얇은 보라색 세로선은 표 경계에 붙이지 않고 순위 열 안쪽에 배치한다. 1~3위 순위색과 섹션 제목의 짧은 보라색 선만 추가 장식 포인트로 사용한다.
- 저장소에서 직접 제공하는 Pretendard Variable 하나로 한글, 영문과 숫자를 통일한다.
- 주 버튼은 후보 웹과 같은 44px 높이와 6px 모서리를 사용한다. 표와 요약의 일반 텍스트는 13px 이상, 메뉴와 주요 숫자는 16~18px로 표시한다.
- 메뉴와 식당의 지도 링크는 평소 데이터 텍스트처럼 보이며 마우스나 키보드로 가리켰을 때만 밑줄을 표시한다.
- 메뉴·식당·추천인 수는 카드로 나누지 않고 얇은 세로 구분선과 숫자·레이블의 크기 차이로 구분한다.
- 장식 요소가 데이터보다 강하게 보이지 않게 한다.
- 작은 화면에서도 가로 스크롤 없이 순위와 핵심 수치를 읽을 수 있어야 한다.
