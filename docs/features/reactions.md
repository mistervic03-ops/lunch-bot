# 후보별 좋아요와 반응 집계

## 목적

구성원이 Slack을 벗어나지 않고 그날의 식당과 메뉴 후보에 가볍게 호감을 표현하고, 장기적으로 사내에서 반응이 좋았던 식당, 메뉴와 추천인을 재미있는 통계로 확인할 수 있게 한다.

## 상호작용

밥라투스트라 메시지 하나에 표시된 후보 순서와 번호 반응을 연결한다.

```text
1번 후보 ↔ 1️⃣
2번 후보 ↔ 2️⃣
3번 후보 ↔ 3️⃣
```

- Slack 메시지 게시 후 봇이 해당 번호 반응을 미리 추가한다.
- 구성원은 마음에 드는 후보 번호를 누른다.
- 여러 후보에 반응할 수 있으며 한 사람당 한 표로 제한하지 않는다.
- 번호 이모지는 브랜드 장식이 아니라 후보를 구분하는 기능적 입력이다.
- 식당별 메시지, 버튼 또는 별도 카드로 분리하지 않는다.

## 수집 방식

상시 Slack 이벤트나 Socket Mode를 사용하지 않는다.

1. 평일 오전 11시 작업이 시작되면 최근 다섯 개의 밥라투스트라 메시지 정보를 추천 로그에서 찾는다.
2. Slack `reactions.get`으로 각 메시지의 번호 반응 수를 조회한다.
3. 봇이 미리 추가한 반응 한 개를 전체 수에서 제외한다.
4. 다섯 메시지의 조회가 모두 성공하면 각 후보 행의 `좋아요 수`와 `좋아요 집계 시각`을 Sheets API 요청 한 번으로 갱신한다.
5. 오늘의 새 추천을 게시하고 번호 반응을 추가한다.

최근 다섯 개 메시지를 반복해서 확인하므로 늦게 추가되거나 취소된 반응도 해당 기간 안에는 다음 실행에서 반영된다. 그보다 오래된 메시지의 집계는 마지막 동기화 값으로 유지한다.

개인별 반응 이력은 저장하지 않는다. Slack 응답의 전체 `count`만 사용하여 개인정보와 데이터 구조를 최소화한다.

다섯 메시지 중 하나라도 조회하지 못하거나 Sheet 갱신이 실패하면 일부 값만 바꾸지 않는다. 운영 채널에 알리고 기존 좋아요 값을 모두 유지한 채 당일 추천 게시를 계속한다.

새 메시지는 게시 직후 구성원의 반응이 아직 없으므로 그 실행에서는 집계하지 않는다. 이후 평일 실행 때 최근 다섯 메시지 범위에 포함되어 집계된다.

## Slack 권한

- `chat:write`: 밥라투스트라 메시지와 운영 알림 게시
- `reactions:write`: 봇의 `1️⃣`, `2️⃣`, `3️⃣` 반응 추가
- `reactions:read`: 기존 밥라투스트라 메시지의 반응 수 조회

Events API, `reaction_added`, `reaction_removed`, Socket Mode 권한은 사용하지 않는다.

## 통계로 해석할 수 있는 값

### 후보를 가장 많이 제보한 사람

`lunch_options`의 유효한 식당과 메뉴 행을 `추천인`별로 센다.

### 가장 많은 반응을 받은 추천인

추천 로그의 `좋아요 수`를 현재 `lunch_options`의 식당, 메뉴와 연결하고 `추천인`별로 합산한다.

### 가장 많은 반응을 받은 식당과 메뉴

- 식당: `식당`별 `좋아요 수` 합계
- 메뉴: `식당 + 메뉴`별 `좋아요 수` 합계

이 값은 절대적인 품질 평가가 아니라 밥라투스트라에 노출된 후보가 받은 Slack 반응 수다. 사용자 문구에서는 `최고의 식당`이나 `가장 좋은 추천인` 대신 `가장 많은 반응을 받은 식당`, `이번 달 점심의 길을 가장 많이 보탠 사람`처럼 실제 측정값을 정확히 표현한다.

## 추천인 집계의 한계

`추천인`은 자유 입력이며 선택값이다.

- 값이 없는 행은 추천인 통계에서 제외한다.
- 앞뒤 공백은 정리하지만 서로 다른 이름 표기는 다른 사람으로 취급한다.
- 정확한 사내 인사 통계나 보상 기준으로 사용하지 않는다.
- 초기에는 구성원이 같은 표시 이름을 일관되게 입력하도록 안내한다.
- 이름 불일치가 실제로 반복될 때만 Slack 사용자 ID나 별도 구성원 목록을 검토한다.

## 자동 통계 게시

좋아요 데이터는 초기 버전부터 수집하지만 주간 또는 월간 결산은 바로 구현하지 않는다. 약 한 달간 실제 반응 사용 여부와 데이터 양을 확인한 뒤 다음 항목을 별도로 결정한다.

- 월간 밥라투스트라 결산을 게시할지
- 어떤 통계를 몇 개 보여줄지
- 개인 추천인 통계를 공개할지
- 통계 게시 시각과 채널

## 아직 결정하지 않은 사항

- 번호 반응 일부를 추가하지 못했을 때의 수동 복구 절차
- 같은 식당과 메뉴가 시트에서 수정되었을 때 과거 추천인의 연결 방식
